apex-pi — apex_distill 双向闭环自进化架构

apex_distill × SKILL Gene Pool × apex_evolver — 借 pi-mono 生态，实现自进化自修复

核心闭环流程

  • ① 任务输入 → apex_search 匹配 SKILL
  • ② 有 SKILL → apex_executor 走成功路径
  • ③ 无 SKILL → 从头执行 + apex_distill 写新 SKILL
  • ④ apex_scoring 评分执行结果
  • ⑤ apex_evolver 低分重写/淘汰
  • ⑥ SKILL Pool 自我优化 → 闭环

SKILL.md 基因格式

  • --- fitness_score: 0.82 ---
  • --- invocation_count: 47 ---
  • --- success_rate: 0.94 ---
  • --- last_used: 2026-06-05 ---
  • --- source_task: SHA(...) ---
  • --- status: active | re_distilling ---
  • ## 执行路径（人类可读）
  • ```步骤 + pi-mono calls```

进化阈值策略

  • THRESHOLD_ACTIVE = 0.6 → 保留
  • THRESHOLD_REDISTILL = 0.3 → 重写
  • fitness < 0.3 → 淘汰（自然选择）
  • 高阈值 → 质量高但进化慢
  • 低阈值 → 进化快但质量波动
  • 推荐初始值 0.4，可动态调整

apex_distill 写 SKILL 逻辑

  • 任务成功（apex_scoring >= 0.6）
  • 提取成功路径 + pi-mono calls
  • 生成 SKILL.md（头部 metadata）
  • 计算 task fingerprint
  • 写入 skills/{fingerprint}.md
  • 初始化 fitness = 当前 score

apex_evolver 淘汰逻辑

  • 每次 apex_scoring 后检查 fitness
  • fitness < 0.3 → 标记 deprecated
  • 标记 deprecated 后 N 次调用机会
  • 仍低于 0.3 → 删除文件
  • 被淘汰 SKILL 的 pattern 进入
  • 黑名单，防止重复无效尝试

借 pi-mono 生态优势

  • ✅ Feishu SDK（无需自研）

  • ✅ pi-mono 团队维护底层稳定性
  • ✅ apex-pi 只维护 apex 层代码
